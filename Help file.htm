Scripts:

The scripts are composed of a limited numbers of commands which you can arrange in any order to perform a

set of predefined tasks on files.

 

In this example, the script does the following:

1.    Opens file

2.    Set EGG channel to ch#0

(this is for single channel wave file. For multi channel PMF files, you set the value 0 to 1

or any other channel where your EGG data is in.)

3.    Invert the EGG channel. Mathematically multiply the data by -1.

4.    Save file.

5.    Close file.

The option column is not implemented yet.

The value in the last column is the parameter that associated with the option for the command. Currently you manually

Enter this value. When the option gets implemented, then the option dialog sets this value.

 

Browse for input files and source folder:

The script will act on all the files in a given folder. All the files are affected. The script does not distinguish between files.

If the file type is not recognized, it will be ignored.

To select a folder, press the “Browse for source folder” button. Then in the selector dialog, DOUBLE CLICK on the folder

you are interested in to view its contents. Once you are satisfied with the source files, and then press the select button.

If you do not perform the last step, a wrong folder is selected.

 

Convert format to Wave:

EggWorks takes as input .pmf or .ptk format files with any number of channels. It outputs a separate single-channel .wav file for each channel in each input file. The .wav format used is 16 bit, which is compatible with the Emu database system. The first channel in an input file is assumed to be audio, and is output as a file named filename_Audio.wav. The next channel in an input file is output as a file named filename_ch1.wav, while the third channel in an input file is output as a file named filename_ch2.wav (etc.). This follows the channel-naming convention of PCquirerX.

 

Changing polarity of EGG file

Some computer sound cards reverse the polarity of EGG signals (IVFCA instead of VFCA). EggWorks can switch the polarity back to VGCA.

The polarity can be switched for many reasons, depending on the hardware design of soundcards or various other settings.

Either PMF or wav files can be used for this script operation.

The predefined scripts are found in the installed folder (typically in C:\Program Files\EggWorks) in a sub folder called “scripts”.

 

There are two predefined scripts available to perform this task on many common files.

1.    scriptWaveInvert.txt

Inverts a single channel wave formatted file, setting the data channel to 0.

Convert->Invert Single Channel Wave File menu invokes this script.

2.    scriptInvert.txt

Inverts the channel#1 of file.

Convert->Invert Ch#1 pmf File menu invokes this script.

 

 

Open the folder containing the input files (so that you see the list of files to be switched) and if desired change the folder for the output files. 

Go runs the script and the list of output files appears.

 

 

EGG Analysis:

EggWorks takes as input either multi channel PMF files (where the second, ch1, is the EGG), or converted single channel wave files containing EGG data.

It outputs a separate text file for each input file, named inoutfilename.egg.

 

Under Script – Create/Open, open an analysis script. Two default scripts can be found in Program Files – EggWorks – scripts.

For .pmf input files, use the script script.txt; for .wav input files, use the script scriptwav.txt. Then open the folder containing the input files

(so that you see the list of files to be analyzed) and if desired change the folder for the output files. 

The script will run on all files in the folder, so either delete any non-EGG files before running the script, or delete their output files after the analysis has run. 

Analysis scripts can be modified (see instructions in script window) and the new versions saved for future use (Save).

The parameters of the analysis are as follows;

EGG:SetChannel    options           sets the data channel where the EGG data is. e.g.       0= only channel in converted wav file

                                                                                                                                                            1=channel#1 for pmf file

                                                                                                                                                            2= channel#2 for pmf file

                                                                                                                                                            …

 

Option:Smooth         options           sets the number of points to smooth the data. e.g.        5= five point smoothing

                                                                                                                                                            12= twelve point smoothing

                                                                                                                                                            100= one hundred point smoothing

                                                                                                                                                            …

Note about smoothing. In general, the EGG data is rough and not very smooth. The smoothing function

will perform a filtering effect on the data with averaging the data over a window with a given number of data points.

5 points does not have a lot of effect but will be satisfactory for most of EGG data.

You may increase or decrease this value to any number you wish.

                                                                                                                                                            …

Currently the options column is not implemented and these parameters have to be entered manually. In the subsequent versions

the Options column will open a dialog box where you can enter these parameters.

 

1.    Scipt.txt

Opens file

Set EGG channel to 1

Sets smoothing to 5 points

Smooth the entire channel#1

Performs EGG analysis on all channel#1

Open a new log file

Enters all the EGG data into the log file and save it

Close file

 

 

2.    scriptwav.txt

Is the same as above, except the input is a single channel wave formatted file and the EGG channel is set to channel#0.

 

3.    ScriptInvert.txt

Inverts the EGG data in channel#1 of a PMF file

 

4.    scriptWaveInvert.txt

Inverts the EGG data in single channel wave formatted file

 

 

About EGG analysis (parameters that EggWorks outputs)

Closing Quotient is calculated by 4 methods

CQ                  Closing Quotient calculated by traditional threshold method (R. J. Baken and R. F. Orlikoff (2000), Clinical Measurement of Speech and Voice, 2nd edition. Singular Publishing. Chapter 10, pp. 426-27, and references cited there; see also http://voiceresearch.free.fr/egg/thresholdmethods.html#EGG).  In EggWorks the threshold is set to 25% and cannot be changed.

CQ_H             Closing Quotient calculated by the “Hybrid” method introduced by Howard, Lindsey & Allen (1990), Toward the quantification of vocal efficiency, J. Voice 4: 205-212, and described by http://voiceresearch.free.fr/egg/thresholdmethods.html#EGGDEGG: the closing peak in the derivative is used as the moment of closing, while the threshold EGG signal (with the threshold set to 3/7) is used for the moment of opening

.

CQ_PM          Closing Quotient calculated from the derivative of the EGG: the closing peak in the derivative is used as the moment of closing, and the opening peak in the derivative is used as the moment of opening. See http://voiceresearch.free.fr/egg/thresholdmethods.html#DEGG

CQ_HT          Closing Quotient calculated by Henry Tehrani’s method: the closing peak in the derivative is used as the moment of closing, and the y-value of the EGG signal is found at the time of that peak. That y-value then is found on the opening phase of the EGG signal, and its time taken to be the time of opening.

Peak Vel                    The max value of the derivative of the EGG signal, in the increasing part of the cycle (so, this is the peak increase in contact, or PIC)

See also A. Michaud’s DECPA measure, http://www.isca-speech.org/archive/sp2004/sp04_633.pdf.

 

Peak Vel Time         The time at which the peak velocity value occurs
